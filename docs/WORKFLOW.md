# Voice Capsule Workflow Guide

## Complete Recording-to-Note Pipeline

This guide shows the exact workflow from recording audio to creating formatted Obsidian notes.

## Workflow Overview

```
Record → Transfer → Transcribe → Edit → Organize → Link
  ↓         ↓          ↓          ↓        ↓        ↓
DJI Mic   USB      通义听悟    Gemini   Obsidian  Knowledge
                                                    Graph
```

## Step-by-Step Process

### Step 1: Recording

**Hardware Setup**:
- Clip DJI Mic 3 to collar or clothing
- Press record button (red light indicates recording)
- Speak naturally, don't worry about perfection

**Recording Tips**:
- Use 12dB gain for quiet environments (mumbling mode)
- Keep mic 10-15cm from mouth
- Record in WAV format for best quality
- Battery lasts 6+ hours

**Use Cases**:
- Meetings (2+ hours)
- Brainstorming sessions
- Post-reading Feynman explanations
- Walking thoughts
- Lecture notes
- Personal reflections

### Step 2: Transfer Files

**Method A: USB Cable** (Recommended)
```bash
# Connect DJI Mic 3 to computer via USB
# Device appears as: /Volumes/DJI MIC 3 (Mac) or E:\ (Windows)

# Files location:
/Volumes/DJI MIC 3/RECORD/

# Copy to local inbox:
cp /Volumes/DJI\ MIC\ 3/RECORD/*.wav ~/voice-capsule/inbox/
```

**Method B: Via iPhone**
1. Connect DJI Mic 3 to iPhone via Bluetooth
2. Open DJI Mimo app
3. Download recordings
4. AirDrop to Mac

### Step 3: Transcribe with Gemini

**One-Step Solution**: Gemini handles both ASR and editing!

**Process**:
```bash
# Use our Python script
python scripts/python/transcribe.py audio.wav --output transcription.txt
```

**What happens**:
1. Upload WAV to Gemini API
2. Gemini transcribes audio (supports 100+ languages)
3. Auto-detects language (Chinese, English, Swedish, etc.)
4. Returns formatted, edited text
5. No separate editing step needed!

**Why Gemini?**
- FREE tier: 60 requests/min, 1500 requests/day
- Excellent ASR quality (multilingual)
- Built-in context understanding
- Single API for everything
- No China-specific services needed

**Cost**: ~$0.35 per 1M tokens (essentially free for personal use)

## Automation Scripts

### Auto-Process New Recordings

```bash
#!/bin/bash
# watch-and-process.sh

INBOX="$HOME/voice-capsule/inbox"
PROCESSED="$HOME/voice-capsule/processed"

# Watch for new WAV files
fswatch -0 "$INBOX" | while read -d "" file; do
    if [[ $file == *.wav ]]; then
        echo "Processing: $file"
        
        # Upload to 通义听悟 (manual step for now)
        echo "Upload to: https://tingwu.aliyun.com"
        
        # Move to processed
        mv "$file" "$PROCESSED/"
    fi
done
```

### Batch Process Transcriptions

```python
# batch_process.py
import os
from pathlib import Path
from transcribe import VoiceCapsuleTranscriber
from obsidian_sync import ObsidianSync

inbox = Path("~/voice-capsule/transcriptions").expanduser()
transcriber = VoiceCapsuleTranscriber()
sync = ObsidianSync()

for txt_file in inbox.glob("*.txt"):
    # Read transcription
    text = txt_file.read_text()
    
    # Create Obsidian note
    note_path = sync.create_note(
        transcription=text,
        audio_filename=txt_file.stem
    )
    
    print(f"Created: {note_path}")
    
    # Archive
    txt_file.rename(f"~/voice-capsule/archive/{txt_file.name}")
```

## Troubleshooting

### Poor Transcription Quality

**Problem**: Many errors in transcription

**Solutions**:
1. Check audio quality (use DJI Mic, not phone mic)
2. Update personal vocabulary list
3. Speak more clearly (not faster, just clearer)
4. Use better LLM for editing (Gemini 3.0 Pro > GPT-3.5)

### Slow Processing

**Problem**: Takes too long to process recordings

**Solutions**:
1. Use 通义听悟 batch upload (multiple files)
2. Process during downtime (overnight)
3. Consider local Whisper for instant results
4. Optimize LLM prompt (shorter = faster)

### Storage Issues

**Problem**: Running out of disk space

**Solutions**:
1. Delete original WAV after transcription
2. Compress archives (zip old recordings)
3. Keep only text, delete audio after 30 days
4. Use cloud storage for backups

## Best Practices

### Recording

- ✅ Test audio before important recordings
- ✅ Use airplane mode to avoid interruptions
- ✅ Charge battery before long sessions
- ✅ Speak naturally, don't perform
- ✅ Record in quiet environment when possible

### Transcription

- ✅ Process within 24 hours (while fresh)
- ✅ Update vocabulary list immediately when errors found
- ✅ Use consistent LLM (don't switch mid-project)
- ✅ Save raw transcription (before editing)
- ✅ Version control your prompts

### Organization

- ✅ Use consistent naming convention
- ✅ Tag notes immediately
- ✅ Link to related notes within 24 hours
- ✅ Weekly review and cleanup
- ✅ Archive old recordings monthly
